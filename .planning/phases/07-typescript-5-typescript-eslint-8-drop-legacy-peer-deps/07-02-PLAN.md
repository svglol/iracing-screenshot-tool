Rewire `eslint.config.js` to use typescript-eslint 8's native flat-config entries via the `typescript-eslint` umbrella's `tseslint.config()` helper. Drop `'prettier'` from the FlatCompat `compat.extends()` chain (D-07: `prettierRecommended` at the end of the array handles prettier-vs-lint conflicts natively). Purpose: Satisfy TS-02's "rule renames migrated in config" clause by swapping the legacy `@typescript-eslint@2` transitive FlatCompat load (via `eslint-config-standard@14`'s plugin list) for native flat-config entries. The `tseslint.config()` helper is MANDATORY per research §Pattern 1 + §Pitfall 1: direct `...tseslint.configs.recommended` spread would leak `@typescript-eslint/parser` into the `base` config's global `languageOptions.parser` and break `@babel/eslint-parser` for every `.js` and `.vue` file.

Output: `eslint.config.js` rewritten with 6 entries (was 5; +1 new tseslint block, 1 edit to drop 'prettier' from FlatCompat chain), and a single commit `refactor(eslint): wire typescript-eslint 8 as native flat-config entries`.
