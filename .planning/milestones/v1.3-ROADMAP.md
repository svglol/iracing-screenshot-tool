## Milestone Goal

Ship user-visible UI polish via isolated, low-risk dependency upgrades — independent of the Vue 2/3 migration track. Both Font Awesome and the Prettier formatter were the Vue-2 ceiling for their respective stacks; this milestone takes them as far as the Vue 2 runtime allows.

## Milestone Summary

### Key Decisions

- **Phase 3 D-04: two-commit bisect shape** — `chore(deps):` + `refactor(icons):` split cleanly isolates dep-bump regressions from template-rename regressions via git bisect. Precedent reused as Phase 4's D-07.
- **Phase 3 D-07: icon library pruning** — After three-grep audit confirmed zero .vue template usage and zero dynamic `:icon=` constructions, 3 unused icons (faUserCog, faInfoCircle, faCamera) removed from `library.add`. Registration went from 11 → 8 icons.
- **Phase 4 Pitfall 4 derogation** — `.prettierrc useTabs: true` clashed with ESLint `standard`'s `no-tabs: error` post-reformat (+368 new errors). User approved wiring `eslint-config-prettier` into `.eslintrc.js` (1-line, minimum-scope fix) now rather than deferring to v1.4's full ESLint-9 flat-config migration. Lint dropped to 1881 (−48 below baseline).
- **`--legacy-peer-deps` is now LOAD-BEARING** — was previously a workaround for typescript-eslint@2 vs eslint@7; Phase 4 added eslint-plugin-prettier@5 peer `eslint >= 8` as a second conflict. Deferred to v1.4 Tooling Modernization.
- **Scope discipline:** `bot/` workspace NOT reformatted (explicit Phase 4 D-01 exclusion; `bot/` was intentionally set up without Prettier in v1.2).

### Issues Encountered

- **wave2-executor / wave3-executor stalls** — In both Phase 3 (Plan 02) and Phase 4 (Plan 03), executors reached blocking checkpoints, returned control, received user resume signals via `SendMessage`, and then did not resume. Orchestrator finalized both commits in-place using plan-locked message templates with real gate numbers + user UAT outcomes. Outcome was byte-identical to the executor's planned path in both cases.
- **Phase 3 baseline path correction** — webpack writes `dist/renderer.js` directly (single file), not `dist/renderer/*.js` as research assumed. Corrected across BASELINE.md + BUNDLE-DIFF.md; plan's label wording preserved verbatim so verify-grep contract matched.
- **Phase 4 HelpModal.vue parser error** — pre-existing `<p>/</p>` orphan in HelpModal template that both Prettier 2.0.2 and 3.3.3 rejected. Wave 1 (`fix(HelpModal)`) unblocked `prettier --check` by removing 2 lines.

### Tech Debt Accumulated (all pre-existing or intentionally deferred)

- FA v5.2.0 CDN `@import` in `src/renderer/assets/style/main.scss:153` — dead-weight from 2020, out of Phase 3 scope per CONTEXT.md; defer to v2.0 Bulma/Buefy work OR standalone `chore(style):` follow-up
- `electron.BrowserWindow.addDevToolsExtension is not a function` at `src/main/index.js:116` — Electron 41 API removal; pre-existing; future main-process cleanup
- `--legacy-peer-deps` load-bearing for npm install — resolves with v1.4 ESLint 9 migration
- `eslint-plugin-prettier` installed but not wired (only `eslint-config-prettier` is) — Pitfall 4 minimum-scope; full integration deferred to v1.4
- Transitive `prettier@2.8.8` via `vue-loader`/`@vue/compiler-sfc` — resolves naturally with v2.0 Vue 3 migration
- Carried-over v1.2 debug sessions: `capture-resolution-no-reshade` (verifying), `gallery-delete-not-removing-file` (awaiting_human_verify)
- 5 pre-existing v1.2-era quick-task artifacts with missing SUMMARY files
