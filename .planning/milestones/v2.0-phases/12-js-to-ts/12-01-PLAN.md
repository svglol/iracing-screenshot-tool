---

### Transform rules

1. **Drop `'use strict';` directive** (line 1 in most files) — TS modules are always strict.

2. **Convert CommonJS `require(...)` to ES `import` where the target has types**:
   - `const path = require('path')` → `import * as path from 'path';` (or `import path from 'path';` if `esModuleInterop` resolves; tsconfig has it true, so default form works)
   - `const { fs, os } = require(...)` destructuring → `import { fs, os } from '...'`
   - **KEEP `require()` CommonJS form for ELECTRON modules** (`const { ipcRenderer } = require('electron')`) because the renderer-side dynamic-require pattern (`if (process.type === 'renderer') { ... require('electron') ... }`) must NOT be hoisted to a top-level static import — it would eagerly load electron in the main process too. Config.ts depends on this.
   - **KEEP `require()` for relative imports** INSIDE conditional blocks or inside functions (e.g., desktop-capture's `require('../utilities/desktop-capture')` lazy-require pattern in main-utils.js — unchanged until Plan 02).

3. **Convert CommonJS `module.exports = { a, b }` to ES `export { a, b };`** at end of file.
   - For `config.ts` (dual export), convert the if/else to:
     ```typescript
     // eslint-disable-next-line @typescript-eslint/no-require-imports
     const { ipcRenderer } = require('electron');
     // Or keep entire existing dynamic-require pattern + `module.exports =` — TS accepts it under esModuleInterop + allowJs inheritance
     ```
     **Simplest safe approach for config.ts**: keep `module.exports =` in both branches (CommonJS-in-TS is legal), add `// @ts-expect-error — dynamic ipcRenderer/Store export shape` on top of each branch ONLY if tsc flags it. Budget 1 `@ts-expect-error` for this file.

4. **Annotate every export's signature** (D-12-08: public API gets types):
   - Function params: primitive → `string | number | boolean`; unknown shape → `unknown`; structurally-typed → inline object type `{ x: number, y: number }`
   - Return types: explicit when non-trivial (boolean returns OK to infer; complex objects → write it out)
   - Optional params with `= default`: add `?` and default as usual — TS infers optionality from default

5. **`any` allowed as escape hatch with TODO comment** (D-12-01 cap ≤ 15 phase-wide, ≤ 3 this plan):
   ```typescript
   // TODO(ts): narrow once Electron type coverage matures
   function serializeDisplay(display: any): { id: number, label: string } | null { ... }
   ```

6. **`@ts-expect-error` format** (D-12-01 prefers over `@ts-ignore`):
