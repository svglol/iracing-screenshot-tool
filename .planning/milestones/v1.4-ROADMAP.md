## Milestone Goal

Modernize the invisible tooling layer (TypeScript, ESLint, Babel ecosystem) before the Vue 3 migration, and resolve tech debt carried over from v1.3. Scope: retire deprecated Babel packages, migrate ESLint 7 → 9 + flat config + full prettier wiring, upgrade TypeScript 3.8 → 5.7 + typescript-eslint 2 → 8, and remove the `--legacy-peer-deps` flag from the install workflow.

## Milestone Summary

### Key Decisions Made

- **D-15 Option 1 (persist --legacy-peer-deps past v1.4) was OBSOLETED** — Phase 6 D-01 Amendment (`eslint-plugin-vue` 6→9) cleared the first peer conflict; Phase 7 TS-02 cleared the second; LINT-03 landed as originally scoped.
- **Phase 6 D-01 Amendment** — `eslint-plugin-vue` 6→9 upgrade escalated mid-phase when v6 crashed under ESLint 9's removed `codePath.currentSegments` API. User approved Option A (upgrade plugin) over Options B/C (disable crashing rules / escape hatch).
- **`@eslint/compat fixupConfigRules`** — added as the bridge for remaining legacy plugins (`eslint-config-standard@14`, `eslint-plugin-node@11`, `eslint-plugin-promise@4`, `@typescript-eslint@2` transitively). Enables ESLint 9 runtime compatibility for plugins not yet flat-config-native.
- **`tseslint.config()` helper MANDATORY** (research Pitfall 1) — direct spread of `tseslint.configs.recommended` would globally override `@babel/eslint-parser` because the base config lacks a `files:` scope. Helper propagates `files: ['**/*.ts']` into every inner config.
- **D-04 bisect shape proven across all three phases** — Phase 5 (four-commit), Phase 6 (two-commit + Phase 7 three-commit chains. Bisecting between HEAD and `chore(deps)` cleanly isolates dep-bump regressions from content changes.
- **Scope control via `.eslintignore` → `ignores:` migration** — Phase 5 established `bot/` + `dist/` exclusion patterns; Phase 6 migrated to flat-config `ignores:` array; Phase 7 preserved without modification.
- **Dual-parser coexistence in flat config** — `@babel/eslint-parser` for `.js/.vue`, `@typescript-eslint/parser` scoped to `files: ['**/*.ts']` only. Currently dormant (zero `.ts` files in `src/`) but wired for future additions.

### Issues Resolved

- Retired deprecated `babel-runtime` + `babel-eslint` (Phase 5)
- Migrated ESLint 7 → 9 flat config (Phase 6)
- Full `plugin:prettier/recommended` wiring, superseding v1.3 Phase 4 Pitfall 4 derogation (Phase 6)
- Upgraded TypeScript 3.8 → 5.7.3 (Phase 7)
- Cleared both peer conflicts that required `--legacy-peer-deps` (Phase 7 + Phase 6 D-01 Amendment)

### Issues Deferred (to v2.0)

- `eslint-plugin-vue` 9 → 10+ (matches Vue 3 major)
- `vue-eslint-parser` 7 → 9
- `eslint-config-standard` 14 → 17+/neostandard migration
- `eslint-plugin-import@2`, `eslint-plugin-node@11`, `eslint-plugin-promise@4`, `eslint-plugin-standard@4` — legacy pins remain via FlatCompat + @eslint/compat shims
- `@babel/eslint-parser` → `@typescript-eslint/parser` as primary for `.js/.vue`
- Converting `.js` → `.ts` in `src/`
- Stricter tsconfig flags (`noUncheckedIndexedAccess`, `exactOptionalPropertyTypes`)
- Jest 25 → Vitest (pairs with Vite+Electron bundler switch)

### Technical Debt Incurred

- Jest globals for `.test.js` files — ~693 `no-undef` firings were visible in Phase 5 baseline addendum but are now outside v1.4 scope (bot/ ignored). Post-v1.4 ticket candidate if Jest env override is desired for readability.
- `@eslint/compat fixupConfigRules` wrap is a compatibility shim for legacy plugins. Will be removed once those plugins upgrade in v2.0.
