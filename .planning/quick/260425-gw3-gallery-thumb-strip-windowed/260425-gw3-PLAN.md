<objective>
Make the gallery thumbnail strip in `Home.vue` larger, windowed (11 visible at a time), centered around the active preview, with smooth slide-and-clamp scrolling and lazy-loaded images.

The current strip:
- Sets `flex: 0 0 calc(100vh / 6)` on each thumb → ~180px wide on 1080p, ~150px tall — not the requested "larger" but visually mid.
- Has `scroll-behavior: smooth` on the strip and a wheel-to-scrollLeft binding (`bindCarouselScroll`).
- Uses `v-lazy` on every indicator's `<img>` → already lazy-loaded.
- Has a red drop-shadow highlight on the active indicator's img.
- Does NOT center the active thumb when the user navigates — the strip stays at whatever scroll position the user last left it at.

The desired strip:
- Sized so exactly 11 thumbs fit the visible viewport (subtract the 240px sidebar).
- Active thumbnail centered in the visible viewport whenever the carousel `selected` index changes, with the browser auto-clamping at start/end (you don't try to scroll past 0 or past `scrollWidth - clientWidth`).
- Lazy-loading: keep the existing `v-lazy` directive — only thumbs that scroll into the viewport actually request image data. With 11 visible at a time, only ~11 (+ a few for IntersectionObserver root-margin) ever load.
- Active highlight more prominent: keep the red drop-shadow, add a 1.05x scale and a smooth transition.

**Why scrollIntoView + flex-sizing instead of conditionally rendering only 11 indicators**:
- Oruga's carousel owns the indicator-item wrapper element; conditionally rendering content inside the slot still leaves the wrapper in DOM, requiring an awkward `:has()` CSS hack to hide outer wrappers.
- `scrollIntoView({ inline: 'center' })` does the centering math and edge clamping in one call — no manual scrollLeft computation, no off-by-one bugs, and it integrates with `scroll-behavior: smooth` for free.
- v-lazy already provides lazy-loading via IntersectionObserver — non-visible thumbs don't request image data even though their `<img>` element exists.
- Net effect from the user's POV is identical to a "render only 11" approach but with simpler code and animated transitions.

Output: edits to `src/renderer/views/Home.vue` only.
</objective>
